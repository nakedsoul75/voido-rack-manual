# Void.O 경량랙 매뉴얼 생성기 — 인수인계 (Handoff)

> 코딩 세션 시작 시 이 문서 + `Void.O_경량랙_조립매뉴얼_생성기.html`를 함께 열면 전체 맥락이 잡힘.

## 1. 한 줄 정의
치수·형태·옵션·문구를 입력하면 **아이소메트릭(ISO) 라인 조립도 + 단계별 매뉴얼**을 결정적으로 생성하는 단일 HTML 웹앱. 모바일 웹 + 인쇄/PDF. 고객 직접 조립용.

## 2. 확정된 결정사항 (변경 금지 기준)
- 브랜드: **Void.O** (COMMANINE 아님)
- 표현: **ISO 입체 라인 도식 + 번호 텍스트**(한글). 도식 안 텍스트 금지(전부 HTML 라벨)
- 구조 정의: **독립형 = 기둥 4개**, **연결형 = 독립형 옆에 기둥 공유로 +2/연**. 일렬 = 독립형1 + 연결형N
- 출력: **모바일 웹페이지**(+인쇄 PDF), 라인 도식
- 운영 방식: **표준 조합 라이브러리 + 미세조정**, 표준 기준은 **납품 3건**
- 독자: 고객 직접 조립

## 3. 현재 완료 상태
- ISO 파라메트릭 엔진(기둥/빔/선반 큐보이드 + 깊이정렬). **기둥 단별 분할로 뒷다리 오클루전 해결**
- 단일/연결(bay) 도식 반영, "기둥 공유 연결" 단계 자동 삽입
- 단계: 구성품→측면프레임→반대쪽→전후면빔→(연결)→선반→(뒷판)→안전핀→(가새)→수평/전도방지→완성/적재
- 옵션: 전도방지·캐스터·뒷판·보강가새 → 단계·BOM·경고 자동 변화
- BOM 자동계산(독립4+연결N×2 표기), 안전/공구/적재/AS 섹션
- 문구 전체 편집 + 자동저장(localStorage)
- **생성 버튼이 폼 값을 직접 읽어 렌더**(이벤트 미발동 환경에서도 동작)
- **모델 DB**: 모델코드 기준 저장/불러오기/새모델/삭제 + 전체 DB JSON 내보내기·가져오기. 표준 모델 3종 내장
- 인쇄 CSS(설정 패널 비인쇄)

## 4. 아키텍처 (코드 맵 — 단일 파일 `Void.O_경량랙_조립매뉴얼_생성기.html`)
- `STATE` (meta/dims/options/contact/text) ← 단일 상태원
- ISO 엔진: `P()` 투영 → `cuboid()` → `render()`(깊이정렬+SVG). `geo()` 치수→부재두께
- `postSegments()` 기둥 단별 분할(오클루전 핵심)
- `rackParts()` 단일 베이, `runParts()` 연결 베이(기둥 공유)
- `DIA{}` 단계별 도식 빌더
- `calcBOM()`, `buildSteps()`, `renderManual()`
- 폼: `buildForm()`/`bindForm()`, 입력 `data-path` → `sp()`로 STATE 갱신
- 생성: `readFormIntoState()` + `renderManual()` = `generate()`
- DB: `getDB/setDB/allModels/refreshModels`, `BUILTIN_MODELS`

## 6. 다음 단계 (마감 작업)

## 7. 권장 스택/세팅
- 현 단일 HTML 유지 가능. 확장 시 **Next.js + Tailwind + Vercel**(실장님 기존 스택)로 이식, ISO 엔진은 순수 함수라 그대로 포팅
- DB는 **Supabase**(모델 레코드 테이블: model PK + JSON state)
